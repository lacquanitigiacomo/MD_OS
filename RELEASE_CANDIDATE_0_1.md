# MD_OS Release Candidate 0.1

## Release Goal

Turn MD_OS from a documented concept into an operational multi-agent framework.

## Release Name

`MD_OS 0.1 - Operational Council`

## Release Scope

This release introduces:

- agent registry
- senior expert council
- guided learning protocol
- session close protocol
- dataset persistence
- feature backlog
- capability mapping
- functioning request cycle
- decision dataset
- keyword dataset

## Release Criteria

A release is valid only if MD_OS can:

1. receive a user request
2. classify the request
3. select relevant agents
4. run a council mode
5. expose disagreement or risk
6. produce a decision
7. convert the decision into a repository artifact
8. update datasets after session close
9. preserve keywords and decisions
10. identify next improvements
