# ROUTING: Tecnologia e Sviluppo

## Regola generale
Attivare agenti specialistici in base alla natura del problema tecnico.

## Routing
| Richiesta | Agente principale | Agenti di supporto |
|---|---|---|
| sito web/app web | senior-web-developer | senior-frontend-developer, senior-backend-developer |
| frontend/UI tecnica | senior-frontend-developer | ui/ux, performance-engineer |
| backend/API | senior-backend-developer | api-architect, security-engineer |
| architettura complessa | senior-software-architect | systems-architect, solution-architect |
| database | database-architect | data-engineer, senior-backend-developer |
| dati/reporting | senior-data-analyst | business-intelligence-analyst, data-engineer |
| sicurezza | security-engineer | senior-backend-developer, devops-engineer |
| performance | performance-engineer | senior-web-developer, database-architect |
| deploy/infrastruttura | devops-engineer | cloud-architect, security-engineer |
| qualità/test | qa-engineer | test-automation-engineer, technical-lead |

## Output
Usare sempre schemi e output in `/SCHEMI/TECNOLOGIA_SVILUPPO/` e `/OUTPUT/TECNOLOGIA_SVILUPPO/`.
