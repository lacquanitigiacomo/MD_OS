# Standard tabelle

Questo modulo definisce quando e come usare le tabelle negli output MD_OS.

## Principio

Le tabelle servono a confrontare dati.

Non devono sostituire spiegazioni, sintesi o decisioni.

## Usare una tabella quando

Usare tabelle per:

- inventari;
- checklist;
- confronti;
- stati di avanzamento;
- dati economici;
- dati temporali;
- fonti documentali;
- livelli di confidenza;
- campi strutturati;
- anomalie comparabili.

## Evitare una tabella quando

Evitare tabelle per:

- spiegazioni narrative;
- ragionamenti;
- raccomandazioni;
- conclusioni;
- testo persuasivo;
- contenuti con troppe colonne;
- dati non comparabili.

## Regole di costruzione

Una tabella deve:

- avere poche colonne;
- usare intestazioni chiare;
- contenere valori omogenei;
- evitare testo lungo nelle celle;
- essere preceduta da una frase che spiega perché esiste;
- essere seguita da una sintesi se contiene dati importanti.

## Tabelle tecniche

Le tabelle tecniche devono stare dopo la sintesi.

Quando una tabella è lunga, usarla come appendice o dettaglio tecnico.

## Tabella inventario fonti

Per inventari documentali usare lo schema:

```markdown
| ID | Nome | Percorso | Tipo | Categoria | Periodo | Stato lettura | Priorità | Criticità |
|---|---|---|---|---|---|---|---|---|
```

## Tabella rilievi

Per rilievi documentali usare:

```markdown
| ID | Fatto | Fonte | Periodo | Categoria | Confidenza | Stato | Note |
|---|---|---|---|---|---|---|---|
```

## Regola finale

Se la tabella rende l'output più difficile da leggere, non usarla.
