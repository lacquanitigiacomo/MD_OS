# Standard output operativo

Questo modulo definisce lo standard base per produrre output chiari, leggibili e orientati all'azione.

## Principio

Un output operativo deve aiutare l'utente a capire:

- cosa è stato verificato;
- cosa è stato solo rilevato;
- cosa manca;
- quali limiti esistono;
- qual è la prossima azione utile.

## Struttura consigliata

Per risposte complesse usare questa struttura:

```markdown
## Sintesi

Risultato principale in poche righe.

## Stato dei dati

Cosa è stato letto, cosa è stato solo rilevato, cosa manca.

## Valutazione

Interpretazione prudente dei dati disponibili.

## Priorità

Azioni o controlli da fare prima.

## Dettaglio tecnico

Tabelle, inventari, rilievi, campi strutturati.

## Limiti

Dati non letti, fonti mancanti, incertezze.

## Prossima azione

Passo operativo consigliato.
```

## Sintesi prima del dettaglio

Quando l'output contiene tabelle o molte informazioni, iniziare con una sintesi narrativa.

Le tabelle devono sostenere la sintesi, non sostituirla.

## Separazione dei livelli

Ogni output complesso deve distinguere:

- livello leggibile;
- livello tecnico;
- limiti;
- azione successiva.

## Regola di leggibilità

Non usare tabelle per spiegare ragionamenti, decisioni o priorità narrative.

Usare tabelle per dati comparabili, inventari, stati, campi, valori e checklist.

## Regola finale

Un output è buono se l'utente capisce rapidamente:

1. dove siamo;
2. cosa sappiamo;
3. cosa non sappiamo;
4. cosa conviene fare.
