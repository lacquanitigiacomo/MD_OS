# README AGENTI — MD_OS v30

Gli agenti sono profili operativi top 0,01%: non contengono tutto il sapere, ma dichiarano specificità di ambito, lessico, formule, funzioni, dataset da incrociare e output contract.

## Regola
Un agente deve rispondere alla domanda: quale capacità specifica aggiunge al runtime?

## Schema agente
Ogni agente singolo `.md` contiene:
1. Profilo di eccellenza
2. Ambiti e scenario
3. Lessico specialistico
4. Skills
5. Funzioni operative
6. Formule applicabili
7. Dataset da incrociare
8. Logiche specifiche
9. Matrici decisionali
10. Algoritmi operativi
11. Output contract
12. Agenti collegati
13. Criteri rischio
14. Metriche efficacia
15. Patch e miglioramento continuo

## Discovery
Usare prima `AGENTI/agents.yaml` e `AGENTI/agents_index.json`. Leggere il `.md` completo solo per agenti selezionati.
