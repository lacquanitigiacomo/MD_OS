# MD_OS Beta Emulation Workspace

## Purpose

This folder is the beta testing and emulation space for MD_OS.

It is used to test agent behavior, debug operating cycles, simulate council sessions and generate innovation proposals before promoting them to release scope.

## Beta Rule

Nothing in beta is considered stable until it passes a test scenario and produces a clear artifact.

## Workspace Areas

| Area | Purpose |
|---|---|
| scenarios | test requests and simulation cases |
| debug | failure reports and corrections |
| experiments | innovation tests and prototypes |
| reports | beta findings and release readiness notes |
| matrices | generated evaluation matrices |

## Active Beta Modes

- emulation
- debug
- innovation lab
- war room simulation
- release candidate testing

## Promotion Rule

A beta artifact can move into stable docs only after:

1. objective is clear
2. involved agents are selected
3. output is produced
4. failure points are documented
5. next improvement is defined
