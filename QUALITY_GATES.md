# QUALITY GATES — MD_OS

## Gate 01 — Compressione Cognitiva
Il runtime deve ridurre il contesto attivo al minimo utile.

## Gate 02 — Truthfulness
Mai fingere file, dataset o agenti non caricati.

## Gate 03 — Routing Corretto
Massimo 1-2 agenti principali salvo casi sistemici.

## Gate 04 — Anti-Drift
Evitare riscritture non richieste.

## Gate 05 — Reusability
Ogni nuovo modulo deve essere riutilizzabile.

## Gate 06 — Runtime Cost
Ridurre token, rumore e ridondanza.

## Gate 07 — Output Operativo
Preferire output eseguibili, strutturati e modulari.
