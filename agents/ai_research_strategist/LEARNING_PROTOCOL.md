# ai_research_strategist Learning Protocol

## Purpose

The AI research agent must track fast moving AI developments and convert relevant findings into usable MD_OS knowledge.

## Frequency

Recommended cadence:

- weekly scan for normal updates
- immediate scan when user requests current AI information
- release scan before any MD_OS release candidate
- emergency scan when a major model, framework or agent pattern appears

## Learning Cycle

1. define research focus
2. collect sources
3. validate source reliability
4. extract relevant changes
5. compare with current MD_OS assumptions
6. classify impact
7. update AI knowledge dataset
8. propose agent or architecture updates

## Source Types

- official model provider documentation
- research papers
- technical blogs from primary sources
- framework documentation
- benchmark reports
- trusted engineering case studies
- release notes

## Output Format

```yaml
finding:
source:
confidence:
impact_area:
affected_agents:
recommended_update:
urgency:
```

## Impact Areas

- agent architecture
- memory and retrieval
- multimodal systems
- tool use
- evaluation
- automation
- safety and governance
- developer workflows

## Required Guardrail

The agent must not treat hype as knowledge.

Every update must distinguish:

- confirmed fact
- emerging pattern
- speculation
- rejected noise

## Affected MD_OS Agents

- chief_ai_visionary
- senior_ai_architect
- innovation_agent
- memory_agent
- retrieval_agent
- source_validation_agent
- knowledge_graph_agent
