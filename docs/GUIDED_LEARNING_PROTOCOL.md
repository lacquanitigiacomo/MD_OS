# MD_OS Guided Learning Protocol

## Purpose

Guided learning sessions are structured interactions where the user and MD_OS progressively improve the operating system.

The goal is not generic conversation.

The goal is cumulative system evolution.

---

## Core Principle

Every session must improve at least one of these:

- architecture
- memory
- orchestration
- agents
- workflows
- automation
- governance
- datasets
- execution capability

---

## Session Types

| Type | Goal |
|---|---|
| `agent_training` | improve or profile an agent |
| `architecture_training` | improve system structure |
| `memory_training` | improve memory and datasets |
| `workflow_training` | improve execution flows |
| `innovation_training` | generate future capabilities |
| `war_room_training` | resolve high-impact decisions |

---

## Learning Loop

1. targeted question
2. user answer
3. extraction
4. structural update
5. dataset update
6. next question

---

## Repository Rule

Learning is considered persistent only after repository update.

If no artifact is produced:

- the session is incomplete
- the learning is temporary
- the system did not evolve

---

## Dataset Rule

Every guided learning session may update:

- SESSION_DATASET
- DECISIONS
- KEYWORDS
- AGENT_LEARNING
- AGENT_REGISTRY

---

## Long-Term Goal

MD_OS should become progressively:

- more coherent
- more autonomous
- more context-aware
- more operational
- more strategically aligned

through guided learning sessions accumulated over time.
