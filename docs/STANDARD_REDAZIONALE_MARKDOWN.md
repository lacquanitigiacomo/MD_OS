# Standard Redazionale Markdown di MD_OS

## Obiettivo

Tutti i documenti ufficiali di MD_OS devono essere scritti in italiano, con struttura chiara, tono tecnico ma leggibile e contenuti completi.

## Regola Lingua

La lingua primaria del repository è l'italiano.

Sono ammessi termini tecnici in inglese solo quando:

- sono nomi di agenti, file, branch, protocolli o comandi
- non esiste un equivalente italiano preciso
- migliorano la leggibilità tecnica

## Struttura Minima dei Documenti

Ogni documento stabile deve contenere:

1. titolo chiaro
2. scopo del documento
3. contesto operativo
4. definizioni principali
5. regole o flussi
6. responsabilità degli agenti coinvolti
7. output attesi
8. criteri di qualità
9. stato del documento
10. prossimi miglioramenti

## Stile

- frasi brevi
- niente riempitivi
- niente introduzioni decorative
- tabelle quando chiariscono relazioni o stati
- esempi concreti quando servono
- sezioni progressive e leggibili

## Qualità Markdown

Usare:

- heading gerarchici corretti
- tabelle ordinate
- liste brevi
- blocchi di codice solo quando utili
- nomi file sempre in backtick
- agenti sempre in backtick

## Tipi di Documento

| Tipo | Scopo |
|---|---|
| Scope release | definire obiettivi, criteri e blocchi di una release |
| Protocollo | descrivere un comportamento ripetibile |
| Dataset | conservare dati strutturati e aggiornabili |
| Profilo agente | descrivere missione, capacità, limiti e output di un agente |
| Report beta | registrare test, problemi e correzioni |
| Matrice | confrontare capacità, rischi, dipendenze o priorità |

## Regola di Completezza

Un file `.md` non deve essere solo una nota.

Deve poter essere letto da una persona esterna e spiegare:

- cosa riguarda
- perché esiste
- come viene usato
- chi lo usa
- quando va aggiornato
